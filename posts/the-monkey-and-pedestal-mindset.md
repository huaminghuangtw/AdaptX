> 「取是能力，舍是境界。」— 電影《破風》

接受、改變、放棄/離開

[Google X](https://x.company/) - 以十倍力衝撞世界最棘手的問題，不是只促成 10％的改善

> _[“Strategic retreat, sacrifice, deciding what to give up is at the core, the very core of success.” — Jensen Huang, NVIDIA’s CEO](https://www.youtube.com/watch?v=oi89u6q0_AY&t=19m57s)_

> _“People think focus means saying yes to the thing you’ve got to focus on. But that’s not what it means at all. It means saying no to the hundred other good ideas that there are. You have to pick carefully.” — Steve Jobs_

也許你跟我一樣，從小就被灌輸「不能輕易放棄、凡事堅持到底」的觀念。久了，我們很容易把「放棄」跟「失敗」綁在一起，即使心裡有股聲音在提醒：這條路好像不太對，我們還是選擇再撐一下、再觀望一陣子。結果就是最珍貴的 [注意力](sherlock-holmes-brain-attic.md) 被消耗在沒有太多未來性的地方。

會不會，我們其實誤解了「放棄」？（放手）

放棄，很多時候是把資源「請回來」，好讓它們能重新分配到真正重要的地方。

放棄，並不是失敗，而是一種重新配置未來機會成本的「技能」。

放棄，是決定去選其他東西。

# Quit 與 Grit 是同一枚硬幣的兩面

相信你一定聽過「堅持，是成功的必需品，是成功的關鍵，是通往成功的道路」之類的心靈雞湯。

堅持（grit）不是壞事；它是幫你把一段旅程走得更完整的能量。只是它需要一個前提：這條路「還值得」。而放棄（quit）是一種調整方向的能力，好讓你不用卡在已經失去意義的迴圈。

我會這樣看：

當痛苦只是「成長期的不適」，而你也能看到一些真實往前的訊號（有人回來、指標慢慢改善、學到的東西持續加深），這時候 grit 很棒。

但如果支撐你留下的主要是：已經花的時間、人家怎麼看你、或是「我都走到這了不走可惜」，那也許可以試著停下來問：我是真的還想選擇它，還是只是被它黏住？

有一個溫和又誠實的檢查法：

> 如果我今天還沒開始這件事，我會用現在的自己，再一次啟動它嗎？

如果答案偏向「可能不會」，也沒關係，這只是代表你成長了、眼界變了。你不是否定過去的自己，而是承認：我現在需要別的東西。

# 為何我們總是「放不下」：那些讓我們黏住的心態

如果你很難對某件事說再見，其實不是你意志不夠，而是人類本來就會被一些心理機制拉住：

1. 沉沒成本：投入越多越不想丟掉，好像放手就否定之前的努力。但未來的你只在乎「接下來能不能更好」。
2. 損失規避：我們天生對「失去」比較敏感，所以常寧願慢慢流血，也不願一次止損。
3. 擁有偏誤：一旦叫「我的專案 / 我的選擇」，我們就會不自覺把它估得更高。
4. 加碼謬誤：狀況不佳時，不是轉彎，而是再加碼，希望用投入感換來意義感。
5. 身份綁定：怕別人覺得自己不夠堅持，於是演給別人看，卻忘了日子是自己在過。

當這些因素同時運作，我們就會把原本應該「平靜調整」的時間點，拖成一個「不得不崩掉才停」的局面。

你可以試著問自己幾個溫柔的小問題：

* 如果 3 個月後一切都差不多，我會不會希望自己今天就先停？
* 我現在做的事情，若明天證明核心假設行不通，這些進度還有意義嗎？
* 我是不是常對自己說「再看看」但其實沒有新的驗證方式？
* 我到底是在前進，還是只是讓自己不要感覺像停下來？

下一段，我想跟你分享「猴子與基座」這個很可愛的比喻，它能幫你更快分辨：我現在做的是關鍵突破，還是其實只是在迴避那隻最難教的猴子。

<!-- Section 4: 猴子與基座 -->

# 猴子與基座：先確認最難的那一塊

我們先聊聊這個可愛又犀利的比喻。想像有人提一個超酷的表演：訓練一隻猴子站上漂亮的基座，然後在台上拋接燃燒的火把。要成功，需要兩件事：

1. 猴子真的學得會雜耍（超級難、關鍵）
2. 有一個漂亮穩固的基座（很容易外包、可預期）

多數人會忍不住先把基座蓋好，因為：做得出進度、好展示、讓人覺得「事情在動」。可是真正決定整個企劃生死的，是那隻猴子能不能被訓練成功。如果猴子最後學不會，再多的燈光、Logo、SOP、Pitch Deck 都只是『好看但無法兌現』。

這就是為什麼我們常覺得「忙了一陣子卻沒真的前進」：因為我們在做的是基座型工作，而不是猴子型驗證。

## 怎麼辨認你現在在做哪一種？

可以這樣簡單區分：

| 類型 | 猴子（核心不確定性） | 基座（可預期、不關鍵） |
| ---- | ------------------ | -------------------- |
| 產品 MVP | 使用者是否願意第一次付費 | Landing Page 視覺、Logo、Notion 文件排版 |
| 創業方向 | 真實目標客群是否有痛到願意換用 | 投資過多時間寫品牌故事 |
| Side Project | 你能否穩定維持每週產出 | 精修一套還沒人看的設計系統 |
| 職涯轉職 | 新職能是否真的能被市場買單 | 重複微調履歷格式 |

一個小提醒：基座不是不重要；它只是「應該在猴子確認可行後」再大量投入的部分。順序錯了，就會被假進度哄睡。

## 三步驟：練習先找猴子

1. 把你正在做的事情寫下來，標記：這件事若未來證明失敗，它是否仍然有價值？若『是』，它很可能是基座。
2. 寫下「如果這件事不成功，最核心的失敗原因可能是什麼？」—— 這通常就是猴子。
3. 設計一個最便宜、最短時間、可部分驗證猴子可行性的行動（例如：一個預購、10 個冷 DM、3 天可跑的技術 spike）。

## 假進度的常見徵兆

* 行動做起來很順手，沒有心理阻力
* 很容易跟別人分享「我們最近做了什麼」
* 延後面對真正不確定的那塊（例如：實際要人掏錢、要陌生人測試、要公開丟出）
* 很忙，但每天結束時沒有新的學習句子可以寫進日誌

你可以問自己：

> 我現在這個小時做的事情，如果核心假設證實不成立，這部分還會被保留嗎？

如果答案是「會啊，至少東西看起來完整」——那八成是基座。

## 一個溫柔的案例對照

| 情境 | 典型做法（基座優先） | 猴子優先的替代做法 |
| ---- | -------------------- | ------------------ |
| 想做知識型電子報 | 先設計 Banner、命名、挑字體、寫歡迎信 | 先手寫 5 封短稿，私訊 10 位朋友請他們誠實回饋是否想看下一封 |
| 想做線上課 | 拍課綱介紹影片、設計投影片母稿 | 先用 1 小時做一場免費 Zoom 微型工作坊看是否有人主動問「之後還有嗎」 |
| 想開產品社群 | 搶 Discord 頻道結構、設定 Bot | 先約 5 個種子用戶開一個 45 分鐘語音討論真正需求 |

## 把「猴子第一」變成習慣

可以試試這個每日 / 每週小檢查：

* 今天最像猴子的那件事是什麼？我有先做它嗎？
* 如果今天不能全部完成，那我是否至少砍下一小塊最不確定的地方？
* 我是否因為怕看見『也許不行』的答案，而拖延這個驗證？

當你越來越習慣先找猴子，你會發現：

* 決策變得更輕盈（因為早點知道不行）
* 更少那種「忙了很久卻說不出成果」的疲憊
* 放棄變得沒那麼刺——因為你投入前就設計了早期檢查點

下一節，我們把視角放到「如果我已經在猶豫要不要停，怎麼好好做一個退出決策？」我會用兩個很簡單的提問陪你釐清。

# 怎麼判斷該不該收手：兩個溫柔但犀利的問題

有時候你不是不知道該做什麼，你只是卡在「怕做錯選擇」的狀態裡。這裡借用 Steven Bartlett 的兩層提問，換成更貼近我們日常對話的版本：

## 問題一：現在讓你想退出，主要是「它很辛苦」還是「它其實已經沒意義」？

先把這兩件事分開：

* 如果只是辛苦（學習曲線、量多、短期沒回饋），那我們再看——這個辛苦背後的回報我還在乎嗎？
* 如果是沒意義（我的價值觀變了、環境變了、即使成功我也不期待），那就可以開始設計一個溫和的退出過程。

延伸小問：

> 假設最理想的「成功版本」真的發生，我會真心覺得值得嗎？

若你回答得不夠肯定，可能已經不是 grit 的議題，而是方向需要調整。

## 問題二：我真的還能做些什麼，讓它「不再那麼糟」嗎？

很多僵局不是走不下去，而是我們其實已經沒有主動槓桿（Levers）可拉，只剩被動等待。

* 如果你列得出 2–3 個你還沒實驗、而且明天就能開始的小調整，那也許值得再給它一個「帶著條件的續約」。
* 如果你想不到，或想到的其實只是換個顏色、再寫一份文件、或是『再撐一陣子看看』——那多半只是延遲決定。

你可以寫一張「再試一次清單」：

| 還想嘗試的槓桿 | 最小實驗形式 | 截止日 | 驗證標誌 |
| ---------------- | ------------ | ------ | -------- |
| 例：付費前問卷 | 10 份冷 DM | 10/15 | ≥3 份留下聯絡資訊 |
| 例：調整定價 | A/B 兩方案 | 10/30 | 轉換率差異 ≥15% |

清單空空的？那也許答案已經出現了。

## 小心第三種狀態：原地內耗

很多人卡在「我不確定要不要停，所以我先什麼都不動」。但事實上，不動本身就是一個高成本選擇：時間繼續流逝、心理空轉，又沒有新訊號輸入。

如果你已經卡超過兩週，可以這樣做：

1. 寫下退出後你最怕失去的三件事（身份、成就感、穩定感）
2. 為每一項列出「其實可以用什麼替代方式獲得」
3. 設一個『決定日』（Decision Day），那天之前只專心跑最後一輪猴子驗證或槓桿實驗

到了那天，就用一句話完成決策：

> 因為（核心判斷），我選擇（繼續 X 個月 / 溫柔收手），接下來我要（轉向 / 回收 / 轉換資源）。

下一節，我們把這種「提前替未來的自己準備的決策節點」寫成一套可複用的 Kill Criteria。

<!-- Section 6: Kill Criteria -->

# Kill Criteria：替未來的自己先鋪好『出口燈光』

很多時候我們不是不知道該停，而是當下情緒太多——疲累、捨不得、還想再證明一點點。這就是為什麼 Kill Criteria（退出準則）有用：你在「還清醒」的時候，幫未來比較糾結的自己寫一張溫柔又清楚的說明卡。

你可以把它想成：『如果出現這些訊號，代表我可以開始收回資源，謝謝這段旅程，然後轉身。』

## 為什麼事前寫比較重要？

當你已經「在裡面」時，判斷會被幾種東西干擾：

* 已投入越多越不想停（沉沒成本）
* 過程中偶爾的小好消息（隨機正回饋）讓你覺得『也許就快要翻盤』
* 身份感綁住，不想別人問『怎麼不做了』

事前寫下來，等於幫當下的主觀情緒設一個邊界。

## 四個基本要素

1. 指標（Metric）：你要觀察的是什麼？（例：有效回覆數、留存率、付費轉換、作品集投遞數）
2. 閾值（Threshold）：多少以下（或以上）代表不符合期待？
3. 時間窗（Time Window）：觀察多久？（例：連續 6 週 / 到 11 月底）
4. 行動（Action）：觸發時做什麼？（例：結束 / Pivot / 暫停 1 個月整備）

## 簡單模板

> 如果在（時間窗）裡，（指標）沒有達到（閾值），且我已經完成（嘗試過的核心實驗），我就會（行動）。

填入示例：

> 如果在 10 月底前，募得的真實付費意向 < 5 人，且我已經跑完 2 場測試工作坊 + 15 個 1:1 訪談，我就會暫停這個課程方向，改去聚焦其中最有反應的『X 子題』再重新設計。

## 不同情境的範例

| 類型 | 範例 Kill Criteria | 備註 |
| ---- | ------------------ | ---- |
| 電子報 | 8 週內開信率 <25% 且點擊率無上升趨勢 → 改為雙週刊並重寫定位 | 先確認是否有嘗試標題 / 發送時段 A/B |
| 線上課原型 | 6 週內 ≥3 場試教仍無人主動詢問正式版 | 『主動詢問』是更真實熱度指標 |
| SaaS MVP | 連續 4 週留存 <15%，且已嘗試 2 次核心 onboarding 改版 | 留存優先於新增使用數 |
| 轉職計畫 | 3 個月內目標職務面試邀請 <3，且已投 30 份客製履歷 | 代表市場回饋不足，需要調整職能敘事 |

## 加上一個『退出腳本』

在觸發條件到的那天，你可以用事先寫好的文字與自己（或團隊）對話：

> 我們達到（或未達到）的數字是：＿＿；這符合（Kill Criteria）裡的條件。代表這條路目前交換不到我們想要的未來。謝謝它教我們的 A、B、C。接下來 2 週，我們做（收尾行動清單），然後在（日期）前完成資源再配置。

這段話會讓退出不再像「失去」，而像「好好結束」。

## 給自己的提醒

* Kill Criteria 不是詛咒，而是護欄
* 你可以更新它，但請記錄修改原因，避免為了延後決定而重寫標準
* 不確定時，請一位你信任、敢講實話的朋友當『退出見證人』（Quitting Buddy）

下一節，我整理了一些可以放進你週 / 月節奏裡的小工具，幫你不斷對焦：我還在養猴子，還是又回去打磨基座了？

<!-- Section 7: 實務工具箱 -->

# 實務工具箱：讓『及時調整』成為你的日常肌肉

這裡放幾個你可以嵌進週 / 月節奏的小儀式，溫和地提醒自己：我還在對焦正確的地方嗎？

## 1. 月初『如果今天才開始』檢視

寫一張清單：列出目前主要正在做的 3–5 件事，逐一問：

> 如果我今天才得知這個想法，我還會啟動它嗎？

若答案是否定或遲疑：再問「是需要微調策略，還是整體方向已過期？」

## 2. Red Team 朋友

找 1–2 個願意說真話的朋友，請他們每月幫你攻擊你的持續理由：

* 你為什麼覺得這還值得？
* 有沒有更高上限的替代選項被你忽略？
* 你最近的進度敘述，是輸出成果還是活動描述？

## 3. 反向成本日誌

每週花 5 分鐘寫：因為沒有退出，我這週付出的隱性代價是什麼？（睡眠、情緒、延後其他專案、錯失學習）—— 讓無形的成本被看見。

## 4. 指標分辨卡：真訊號 vs. 假訊號

| 真訊號（可牽動去留） | 假訊號（容易自我感覺良好） |
| ---------------------- | ---------------------------- |
| 留存 / 付費意向 / 再次主動互動 | 點讚數、視覺精緻度、文件頁數 |
| 使用者願意帶朋友進來 | 社群被動加入但低互動 |
| 願意為下一步預留時間 | 只有口頭說「不錯」 |

## 5. 退出腳本（溫柔版）

當到達 Kill Criteria 或你已經很累時，可以念給自己聽：

> 我不是在否定努力，而是在保護未來。我選擇把這段經驗留下，把持續投入的權利收回。接下來我會（收尾行動），然後留白幾天，讓空間長出新的東西。

## 6. 週『猴子優先』排程

在排新一週行程前，先寫下：『如果這週只允許我做 3 件真正關鍵、最不確定的事情，它們是什麼？』先把這些放進行事曆，再安排其他維護型任務。

## 7. 決策後復盤小卡

退出或堅持後，用 5 行紀錄：

1. 我看到的關鍵訊號是？
2. 我採用 / 放棄的假設是？
3. 這次做得好的地方？
4. 下次可以更早做的是？
5. 下一步第一個 30 分鐘行動？

# 結語：你可以用更溫柔的方式往前

很多時候我們捨不得放下一條路，不是因為它真的還有價值，而是我們不確定：如果我放下，我還會不會一樣值得、一樣被肯定。其實會的，甚至更會——因為你開始為自己做更合適的選擇。

今天如果只能做一件小事，試試這個 10 分鐘練習：

1. 寫下一件你心裡反覆猶豫的事
2. 寫出它的『猴子』是什麼，明天用最小單位測一次
3. 幫自己寫一行暫時的 Kill Criteria（時間 + 指標 + 閾值 + 行動）
